Elements Interface

This section describes the interfaces for different element types.

ONSAS.Elements — Module

Module defining element formulations implemented. Each element consists of a data type with the geometric properties such as nodes, cross-sections and a label into its fields.

source
ONSAS.Elements.AbstractElement — Type

Abstract supertype for all elements. An AbstractElement object facilitates the process of evaluating: - The internal forces vector and its tangent matrices. - The inertial forces vector and its tangent matrices. - The mechanical stresses and strains.

Common methods:

  • coordinates
  • dimension
  • dofs
  • local_dofs

These methods is a hard contract and must be implemented to define a new element.

  • local_dof_symbol
  • label
  • nodes
  • weights
  • Base.:∈

This method is a hard contract and for static analysis must be implemented to define a new element.

  • internal_forces

This method is a hard contract and for dynamic analysis must be implemented to define a new element.

  • inertial_forces

Common fields:

  • nodes
  • label

ONSAS.Elements.AbstractEntity — Type

Abstract supertype for all Faces and Elements.

An AbstractEntity object is an entity defined by dofs and node/s with certain coordinates and dimension.

Common methods:

  • create_entity
  • coordinates
  • dimension
  • dofs

Common fields:

  • nodes
  • label

ONSAS.Elements.AbstractFace — Type

Abstract supertype for all elements.

An AbstractFace object facilitates the process of adding boundary conditions on a surface.

Common methods:

  • area
  • create_entity
  • coordinates
  • dofs
  • label

ONSAS.Elements.Node — Type

A Node is a point in space. The coordinates of the node are stored using SVector data type from static arrays.

Fields:

  • dofs – stores the node degrees of freedom, maps symbol to dofs.

ONSAS.Elements.internal_forces — Method

Returns the internal force of a Tetrahedron element t doted with an LinearIsotropicMaterial m.

Arguments

  • material: IsotropicLinearElastic type, the linear elastic material of the tetrahedron element.
  • element: Tetrahedron type, the tetrahedron element for which internal forces are to be computed.
  • displacements: AbstractVector type, the nodal displacements of the element.

Returns

A 4-tuple containing:

  • forces: Symmetric type, the internal forces of the tetrahedron element.
  • stiffness: Symmetric type, the stiffness matrix of the tetrahedron element.
  • stress: Symmetric type, the Cauchy stress tensor of the tetrahedron element.
  • strain: Symmetric type, the strain tensor of the tetrahedron element.
